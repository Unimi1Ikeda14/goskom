# GosKom — Цифровая платформа коммунальных услуг

**GosKom** — это динамическое веб-приложение, представляющее собой единую экосистему для управления жилищно-коммунальными услугами. Платформа призвана автоматизировать взаимодействие жителей с управляющими компаниями, предоставляя инструменты для моментальной оплаты счетов, передачи показаний приборов учета и мониторинга состояния сетей в реальном времени. Основная цель проекта — сделать сферу ЖКХ прозрачной и доступной, избавляя пользователя от необходимости личного посещения офисов и бумажной волокиты.

**Целевая аудитория:** Собственники и арендаторы жилья, стремящиеся к цифровому контролю расходов, а также сотрудники диспетчерских служб, которым необходим оперативный канал информирования граждан об аварийных ситуациях.

### Ключевой функционал
* **Централизованный биллинг:** Автоматическое формирование счетов и удобный просмотр истории начислений.
* **Передача показаний:** Интеллектуальная форма подачи данных счетчиков с валидацией (сравнение с предыдущим периодом).
* **Мониторинг инфраструктуры:** Список и карта актуальных отключений (свет, вода, газ) с фильтрацией по адресу.
* **Инфо-центр:** Лента официальных новостей ЖКХ и уведомлений от администрации.
* **Техническая поддержка:** Система подачи заявок на ремонт и отслеживание их статуса в режиме онлайн.

## 2. Технологический стек

### Backend & Frontend (Monolith SSR)
* **Java (Spring Boot):** Выбран в качестве основного стека из-за высокой надежности и мощных инструментов безопасности (Spring Security). Строгая типизация минимизирует ошибки в финансовых расчетах.
* **Thymeleaf:** Серверный шаблонизатор, позволяющий динамически генерировать HTML-страницы. Это упрощает разработку монолитного приложения, обеспечивая бесшовную передачу данных из контроллеров в интерфейс без необходимости настройки сложного API.

### База данных
* **SQLite:** Компактная реляционная СУБД. Идеальна для данного этапа разработки, так как не требует настройки сервера, хранит данные в одном файле и поддерживает все необходимые SQL-стандарты для работы со счетами и показаниями.

### Инфраструктура
* **Maven:** Сборка проекта и управление зависимостями.
* **Spring Data JPA:** Для абстракции над SQL-запросами и удобного маппинга сущностей базы данных на Java-объекты.

## 3. Визуальное проектирование (Текстовые прототипы)

### Главная страница (`/`)
* **Шапка:** Логотип "GosKom", меню (Новости, Отключения, Счета), кнопка "Личный кабинет" (если авторизован) или "Войти".
* **Блок 1 (Hero):** Заголовок "Управляйте домом в один клик" и кнопка "Оплатить счет".
* **Блок 2 (Виджет):** Список из 3-х актуальных новостей (данные из модели `News`).
* **Футер:** Контакты диспетчерской, ссылки на соцсети.
* **Динамика:** Отображение персонального приветствия, если пользователь в системе.

### Страница "Мои счета" (`/billing`)
* **Блок 1:** Общая сумма задолженности по всем счетам.
* **Блок 2 (Таблица):** Список квитанций. Столбцы: Месяц, Сумма, Статус (кнопка "Оплатить" или текст "Оплачено").
* **Состояния страницы:**
    * *Загрузка:* Спиннер или пустая таблица с надписью "Загрузка...".
    * *С данными:* Список счетов пользователя.
    * *Ошибка:* Сообщение "Не удалось получить список счетов. Попробуйте позже".

### Передача показаний (`/meters`)
* **Блок 1:** Выбор прибора (ХВС, ГВС, Электроэнергия).
* **Блок 2 (Форма):** Поле для ввода цифр, дата замера, кнопка "Отправить".
* **Динамика:** Через `th:errors` выводится ошибка, если введенное значение меньше того, что уже есть в базе.

## 4. Схема данных и маршрутизация

### Модели данных (Сущности)

1. **User (Пользователь)**
   * `id` (Long, PK)
   * `username` (String, Unique) — email.
   * `password` (String) — хеш-пароль.
   * `role` (String) — USER или ADMIN.
2. **Bill (Счет)**
   * `id` (Long, PK)
   * `userId` (FK) — владелец.
   * `amount` (Double) — сумма.
   * `period` (String) — например, "Март 2026".
   * `isPaid` (Boolean).
3. **MeterReading (Показание)**
   * `id` (Long, PK)
   * `type` (Enum) — WATER, GAS, ELEC.
   * `value` (Double).
   * `readingDate` (LocalDateTime).

### Ключевые маршруты (Контроллеры)

**AuthController:**
* `GET /login` — отображение формы входа.
* `POST /register` — обработка данных регистрации.

**BillingController:**
* `GET /billing` — получение списка счетов из БД и передача в модель Thymeleaf.
* `POST /billing/{id}/pay` — обновление статуса `isPaid = true` в SQLite.

**MeterController:**
* `GET /meters` — отображение формы подачи показаний.
* `POST /meters/add` — сохранение нового объекта `MeterReading` с валидацией.

**NewsController:**
* `GET /news` — получение всех новостей.
* `GET /news/{id}` — детальный просмотр новости.
