##GosKom
##1. Концепция и целевая аудитория
 * Название: GosKom
 * Описание: Единая информационная платформа для взаимодействия жителей города с коммунальными службами. Приложение позволяет в реальном времени отслеживать график плановых и аварийных отключений (вода, свет, газ), узнавать актуальные тарифы и передавать показания приборов учёта. Основная цель — сделать процесс оплаты и мониторинга коммунальных расходов прозрачным и доступным в один клик.
 * Целевая аудитория: Владельцы жилья, арендаторы квартир и представители управляющих компаний.
 * Ключевой функционал:
   * Мониторинг отключений по адресу пользователя.
   * Оплата счетов и просмотр истории платежей.
   * Подача показаний счётчиков.
   * Калькулятор стоимости услуг по текущим тарифам.
##2. Структура приложения (разделы)
Согласно требованиям, приложение разделено на 8 логических блоков:
 * Главная (/) — Публичная страница с общими новостями ЖКХ и поиском по базе знаний.
 * Аутентификация (/auth/*) — Единый раздел для входа, регистрации новых пользователей и восстановления пароля.
 * Карта отключений (/outages) — Интерактивный список или карта с текущими работами по городу (динамический список).
 * Тарифы (/tariffs) — Актуальные расценки на услуги для разных групп населения.
 * Личный кабинет (/profile) — Приватный раздел с данными пользователя и привязкой лицевых счетов.
 * Мои платежи (/payments) — Список прошлых оплат с возможностью выгрузки чеков в PDF.
 * Передача показаний (/counters) — Форма для ввода данных счётчиков и график потребления ресурсов.
 * Обратная связь (/support) — Система тикетов для подачи заявок в аварийную службу или ЖЭС.
##3. Технологический стек
Для реализации выбран следующий стек технологий:
 * Frontend: React (библиотека с компонентным подходом), React Router (для навигации) и Redux Toolkit (для управления состоянием корзины платежей и данных пользователя).
 * Backend: Java + Spring Boot. Выбран из-за высокой надёжности, строгой типизации и отличной поддержки работы с БД.
 * База данных: SQLite. Оптимальна для этапа разработки и прототипирования за счёт отсутствия необходимости в сложном сервере БД.
 * Инфраструктура: Развертывание frontend-части на Vercel/GitHub Pages, backend-части — на локальном или облачном сервере с использованием Maven.
##4. Схема данных и API
Основные модели данных:
 * User: ID, ФИО, Email, Пароль (хэш), Адрес, Лицевой счет.
 * UtilityBill: ID, UserID, Тип услуги (свет/вода), Сумма к оплате, Дата начисления, Статус (оплачено/задолженность).
Ключевые эндпоинты API:
 * POST /api/auth/register — Регистрация нового жильца.
 * GET /api/bills/user/{id} — Получение списка всех счетов пользователя.
 * POST /api/counters/submit — Отправка новых показаний счётчиков.
 * GET /api/outages/current — Получение списка активных отключений.
