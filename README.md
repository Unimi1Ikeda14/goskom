# GosKom
## 1. Концепция и целевая аудитория
* **Название приложения:** «GosKom» (Государственные Коммуникации).
* **Краткое описание:** Единая цифровая платформа для автоматизации взаимодействия граждан с коммунальными службами. Приложение позволяет в реальном времени отслеживать графики отключений ресурсов (вода, свет, газ), узнавать актуальные тарифы, передавать показания приборов учёта и просматривать историю платежей.
* **Целевая аудитория:** Жители многоквартирных и частных домов, арендаторы жилья, а также сотрудники управляющих компаний.
* **Ключевой функционал:** Мониторинг аварийных работ, онлайн-просмотр задолженностей, подача показаний счетчиков и система обращений в ЖЭС.

---

## 2. Структура приложения
Приложение состоит из 9 логических разделов:
1.  **Главная (/)** — информационный дашборд.
2.  **Аутентификация (/auth/*)** — вход, регистрация и восстановление (считается как 1 раздел).
3.  **Карта отключений (/outages)** — список текущих и плановых работ.
4.  **Справочник тарифов (/tariffs)** — информация о стоимости услуг.
5.  **Личный кабинет (/profile)** — персональные данные пользователя.
6.  **Мои платежи (/payments)** — история транзакций и архивы чеков.
7.  **Приборы учета (/counters)** — интерфейс для передачи показаний.
8.  **Техподдержка (/support)** — система тикетов для заявок.
9.  **Страница ошибки (404)** — обработка некорректных адресов.

---

## 3. Технологический стек
* **Backend:** **Java + Spring Boot**. Выбран за счет высокой производительности и отличной интеграции с инструментами разработки.
* **Frontend:** **Thymeleaf (Server-Side Rendering)**. Использование шаблонизатора на стороне сервера позволяет быстро генерировать динамический HTML-контент, используя мощь Java-классов напрямую в разметке. Для интерактивности отдельных элементов (валидация, маски ввода) используется **Vanilla JavaScript**.
* **База данных:** **SQLite**. Позволяет хранить данные в локальном файле, что идеально подходит для этапа прототипирования и выполнения учебных проектов.
* **Сборка:** **Maven**.

## 4. Визуальное проектирование 
### 1. Страница: Главная (/)
* **Тип доступа:** публичный
* **Основные блоки:**
    1.  Шапка: логотип GosKom, меню (Услуги, Тарифы, Новости), кнопка «Войти».
    2.  Поисковая строка: «Проверьте свой адрес на отключения».
    3.  Секция «Важное»: 3 карточки с актуальными новостями ЖКХ города.
    4.  Футер: контакты диспетчерских служб.
* **Динамический контент:** Блок 3 (новости загружаются из БД через контроллер).

### 2. Страница: Аутентификация (/auth/*)
* **Тип доступа:** публичный
* **Основные блоки:**
    1.  Центральная форма с переключателем «Вход» и «Регистрация».
    2.  Поля: Email, пароль, номер лицевого счета (для регистрации).
    3.  Кнопка: «Подтвердить».
* **Динамический контент:** вывод сообщений об ошибках при неверном вводе данных.

### 3. Страница: Карта отключений (/outages)
* **Тип доступа:** публичный
* **Основные блоки:**
    1.  Фильтр: выбор категории (Электричество / Вода / Отопление).
    2.  Список: карточки с адресами, временем начала и плановым временем завершения работ.
* **Динамический контент:** блок 2 (список формируется на сервере исходя из актуальных данных в базе).

### 4. Страница: Тарифы (/tariffs)
* **Тип доступа:** публичный
* **Основные блоки:**
    1.  Инфо-блок: текущие законодательные акты о тарифах.
    2.  Таблица: наименование услуги — цена за единицу (м³, кВт/ч и т.д.).
* **Динамический контент:** данные таблицы (цены подтягиваются из сущности `Tariff`).

### 5. Страница: Личный кабинет (/profile)
* **Тип доступа:** приватный
* **Основные блоки:**
    1.  Боковое меню: Мои данные, Платежи, Счетчики.
    2.  Карточка пользователя: ФИО, привязанный адрес, текущий баланс.
* **Динамический контент:** баланс пользователя и его персональные данные.

### 6. Страница: Мои платежи (/payments)
* **Тип доступа:** приватный
* **Основные блоки:**
    1.  Таблица: Дата, Сумма, Статус оплаты, кнопка «Скачать чек».
* **Динамический контент:** история транзакций, отфильтрованная для конкретного `User`.

### 7. Страница: Приборы учета (/counters)
* **Тип доступа:** приватный
* **Основные блоки:**
    1.  Форма: список всех счетчиков по адресу пользователя.
    2.  Поля ввода: «Текущее показание» для каждого прибора.
    3.  Кнопка: «Сохранить данные».
* **Динамический контент:** список счетчиков (зависит от того, какие приборы зарегистрированы за пользователем).

### 8. Страница: Техподдержка (/support)
* **Тип доступа:** приватный
* **Основные блоки:**
    1.  Список «Мои обращения» со статусами (В обработке / Выполнено).
    2.  Кнопка «Создать заявку».
* **Динамический контент:** актуальные статусы заявок из таблицы `SupportTicket`.

### 9. Страница: Ошибка 404
* **Тип доступа:** публичный
* **Основные блоки:**
    1.  Графический блок «404».
    2.  Текст: «Страница портала GosKom не найдена или еще не создана».
    3.  Кнопка: «На главную».
* **Динамический контент:** отсутствует.

## 5. Схема данных и API

**Основные модели данных:**
* **User:** `id, email, passwordHash, address, billing_account`.
* **Counter (Счетчик):** `id, userId, type (water/electricity), currentReading, lastUpdate`.

**Ключевые эндпоинты API :**
* `POST /auth/register` — регистрация.
* `POST /auth/login` — вход.
* `GET /counters` — получение списка счетчиков пользователя.
* `POST /counters/update` — обновление показаний.
