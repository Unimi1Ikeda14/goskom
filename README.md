# GosKom — Единый портал коммунальных услуг

**GosKom** — это динамическое веб-приложение, предназначенное для упрощения взаимодействия жителей с коммунальными службами. Платформа позволяет пользователям быстро оплачивать счета, передавать показания приборов учета и оперативно получать информацию о состоянии сетей в своем доме. Система объединяет финансовые операции, подачу заявок в техподдержку и актуальную ленту новостей об отключениях в одном интерфейсе.

**Целевая аудитория:** Собственники и арендаторы жилья, а также сотрудники управляющих компаний, которым необходим эффективный канал информирования граждан.

## Ключевой функционал

* **Управление платежами:** Просмотр текущей задолженности и имитация оплаты счетов онлайн.
* **Передача показаний:** Форма для отправки данных счетчиков с сохранением истории в базе данных.
* **Мониторинг ресурсов:** Список плановых и аварийных отключений с фильтрацией по адресу.
* **Информационный центр:** Лента новостей ЖКХ и важные уведомления от городских служб.
* **Поддержка пользователей:** Создание заявок (тикетов) на техническое обслуживание.

## Структура приложения (12 разделов)

1.  **Главная страница (/)** — публичная: лендинг с описанием сервиса и блоком свежих новостей.
2.  **Аутентификация (/login, /register)** — раздел для входа и регистрации. Доступен только неавторизованным пользователям.
3.  **Личный кабинет (/profile)** — приватный: редактирование данных пользователя и управление адресами.
4.  **Мои счета (/billing)** — приватный: список квитанций и их текущие статусы.
5.  **Детализация счета (/billing/{id})** — приватная: подробная расшифровка начислений за выбранный период.
6.  **Приборы учета (/meters)** — приватный: интерфейс для ввода текущих показаний счетчиков.
7.  **История показаний (/meters/history)** — приватный: архив всех переданных данных в виде таблицы.
8.  **Новости ЖКХ (/news)** — публичная: общая лента новостей и объявлений.
9.  **Просмотр новости (/news/{id})** — публичная: страница с полным текстом статьи.
10. **Карта и список отключений (/outages)** — публичная: информация о текущих работах на сетях.
11. **Центр поддержки (/support)** — приватный: список обращений пользователя.
12. **Создание заявки (/support/create)** — приватный: форма для описания бытовых проблем (протечки, отсутствие услуг).

## Технологический стек

### Backend & Frontend (Monolith)
* **Java (Spring Boot)** — выбран в качестве основного языка и фреймворка из-за строгой типизации и высокой надежности. Spring Boot ускоряет разработку благодаря автоконфигурации и встроенному серверу приложений.
* **Thymeleaf** — шаблонизатор для серверного рендеринга (SSR). Позволяет создавать динамические страницы на стороне сервера, что упрощает интеграцию данных из Java-кода в HTML. Это избавляет от необходимости настройки отдельного REST API для фронтенда на начальном этапе.

### База данных
* **SQLite** — компактная встраиваемая реляционная СУБД. Она идеально подходит для данного проекта, так как не требует установки отдельного сервера базы данных, хранит все данные в одном файле и обеспечивает достаточную производительность для работы одного пользователя или небольшой группы.

### Инфраструктура
* **Локальная разработка** — приложение запускается локально, база данных SQLite инициализируется автоматически при старте приложения.

## Визуальное проектирование интерфейса

### Главная страница "/"
* **Шапка:** Название "GosKom", навигация по сайту, кнопка входа или имя пользователя (через `th:if`).
* **Блок приветствия:** Краткое описание и призыв передать показания.
* **Динамический контент:** Список последних новостей, который рендерится на сервере с помощью Thymeleaf (`th:each`).

### Мои счета "/billing" (Приватный)
* **Контент:** Таблица со списком счетов. Статус оплаты подсвечивается разными цветами (красный — долг, зеленый — оплачено).
* **Состояния:** Если данных нет, отображается блок "У вас пока нет начислений".

### Передача показаний "/meters" (Приватный)
* **Форма:** Поля для ввода цифр. При нажатии "Отправить" данные уходят на POST-контроллер.
* **Динамика:** Отображение сообщений об успешной отправке или ошибках валидации через атрибуты модели Spring.

## Схема данных и API (Контроллеры)

### Модели данных
* **User (Пользователь):** `id`, `username`, `email`, `password_hash`, `role`.
* **Account (Лицевой счет):** `id`, `userId`, `address`, `account_number`.
* **Bill (Счет):** `id`, `accountId`, `amount`, `period`, `is_paid`.
* **MeterReading (Показание):** `id`, `accountId`, `type` (water/gas/light), `value`, `created_at`.

### Основные маршруты (Маршрутизация)
* `GET /register`, `POST /register` — регистрация.
* `GET /login` — страница входа.
* `GET /billing` — получение списка счетов из БД и рендеринг страницы.
* `POST /meters/submit` — обработка формы отправки показаний.
* `GET /news/{id}` — получение новости по ID и отображение через шаблон.
* `GET /outages` — отображение актуальных данных об отключениях.
